# Screenshots

Drop the screenshots referenced by the main `README.md` / `README.pt-BR.md` here.
Use the exact filenames below so the README links resolve, then uncomment the
image lines in the READMEs.

## Recommended shots

| Filename | What to capture | Why it matters |
|---|---|---|
| `roadmap.png` | The main `/mtrack` window with a healthy list (filters at default) | The hero shot — shows the whole value at a glance |
| `glow.png` | A row (or a few) with the green **obtainable-now** glow | The killer feature: "you can get this right now" |
| `row.png` | A close-up of one row: icon, name, vendor(s), zone + `Cost: N 🪙 Currency (have X)`, button box | Shows the rich per-mount detail |
| `expansion-filter.png` | The **Expansion** dropdown open | Shows filtering by expansion |
| `owned-toggle.png` | List with **Show owned** ticked (owned = colored, missing = gray) | Shows the owned/missing visual coding |
| `minimap.png` | The minimap button + its tooltip | Shows the launch button |

## Tips

- Use a character with **mixed progress** (some renown/rep partway) so you get a
  good mix of statuses: 🟢 Ready, 🟡 Need currency, 🟠 Need requirement, 🔵 Farmable.
- Crop tightly and keep a consistent width (e.g. ~900px) for a clean README.
- PNG preferred. Avoid capturing personal account info if any is visible.

After adding files, update the **Screenshots** section in both READMEs, e.g.:

```markdown
![Roadmap window](images/roadmap.png)
![Obtainable-now glow](images/glow.png)
```
